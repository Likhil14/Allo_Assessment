-- Key design decisions encoded in the schema:
--
-- 1. Inventory.reservedUnits — denormalized counter updated atomically within
--    the same transaction as reservation mutations. Avoids expensive COUNT joins
--    on hot availability-check paths.
--
-- 2. Inventory.version — optimistic lock field. Incremented on every write.
--    Not used for the primary locking strategy (we use FOR UPDATE) but available
--    for future audit/event sourcing use cases.
--
-- 3. Indexes:
--    - Reservation(status) — cron expiry query: WHERE status = 'PENDING'
--    - Reservation(expiresAt) — cron expiry query: WHERE expiresAt < NOW()
--    - Reservation(productId, warehouseId, status) — composite for availability checks
--    - Inventory(productId, warehouseId) — enforced UNIQUE, also serves as index
--
-- 4. Foreign key constraints with RESTRICT on reservations prevent
--    accidental product/warehouse deletion while active reservations exist.
